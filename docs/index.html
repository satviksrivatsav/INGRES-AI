AI & Machine Learning

ML Models & AI Integration

Two custom Scikit-learn models trained on Indian agricultural and water quality datasets, plus Google Gemini for conversational AI.

💧
Resource Planner Model
resource_planner_model.pkl

Predicts water requirements for crops based on land size, crop type, and season. Trained on resource_planner_dataset.csv.

Input: land_size Input: crop_type Input: season Output: water_requirement (L) Output: risk_level
🔬
Safety Checker Model
safety_checker_model.pkl

Classifies water quality as Safe / Warning / Unsafe based on source type and location. Trained on safety_checker_dataset.csv.

Input: source_type Input: location Input: usage Output: Safe Output: Warning Output: Unsafe

🤖 RAG Chatbot Architecture

The chatbot uses a Retrieval-Augmented Generation (RAG) architecture to ensure factually accurate, non-hallucinated responses about groundwater data.

  • ✓ User query parsed for district name and year
  • ✓ Exact GroundwaterData record retrieved from DB via Django ORM
  • ✓ Retrieved facts (extraction %, recharge rate, status) injected into Gemini prompt
  • ✓ Gemini AI explains the data in natural language
  • ✓ Conversation logged to ChatConversation model
  • ✓ Graceful fallback if no district data found
RAG Pipeline

User: "Guntur groundwater 2022"

↓ Parse → district=Guntur, year=2022

↓ ORM: GroundwaterData.objects.get(
  district="Guntur",
  year=2022
)

↓ Context: extraction=78%,
  recharge=65 mm/yr

↓ Gemini prompt + context

→ "Guntur district in 2022 had a
  78% extraction rate indicating...
  conservation advice: ..."
Getting Started

Setup & Installation

Get INGRES-AI running locally in a few minutes.

1

Clone & Create Virtual Environment

Clone the repository and set up an isolated Python environment.

git clone https://github.com/satviksrivatsav/INGRES-AI.git cd INGRES-AI python -m venv venv source venv/bin/activate # Windows: venv\Scripts\activate
2

Install Dependencies

Install all pinned Python packages from req.txt.

pip install -r req.txt
3

Configure Environment

Create a .env file in the project root with your secrets.

SECRET_KEY=your-django-secret-key DEBUG=True AI_API_KEY=your-google-gemini-api-key EMAIL_HOST_USER=your-gmail@gmail.com EMAIL_HOST_PASSWORD=your-app-password
4

Apply Migrations & Seed Data

Create the database schema and populate initial data.

python manage.py migrate python seed_groundwater.py python seed_schemes.py
5

Run Development Server

Start the Django development server and open your browser.

python manage.py runserver # Visit: http://127.0.0.1:8000

🤖 Train ML Models (Optional)

Pre-trained .pkl files are excluded from Git (see .gitignore). To train from scratch:

  1. Open WATER.ipynb in Google Colab
  2. Upload resource_planner_dataset.csv and safety_checker_dataset.csv
  3. Run all cells to train & export .pkl files
  4. Place the generated files in portal/ml_models/

🔐 Required Environment Variables

  • SECRET_KEY — Django secret key
  • DEBUG — True for dev, False for prod
  • AI_API_KEY — Google Gemini API key
  • EMAIL_HOST_USER — Gmail address for OTP
  • EMAIL_HOST_PASSWORD — Gmail app password

🌍 Internationalization

Compile translation messages for Hindi and Telugu support:

django-admin compilemessages

Locale files are in /locale/. Language selection uses a cookie-based switcher in the UI.
